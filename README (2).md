# 🎭 Emotion Classifier — Fine-Tuned DistilBERT

A text classification project that fine-tunes **DistilBERT** on the [dair-ai/emotion](https://huggingface.co/datasets/dair-ai/emotion) dataset to detect **6 emotions** in short text — sadness, joy, love, anger, fear, and surprise — plus an interactive **Gradio demo** you can deploy live.

> Built as a portfolio project to demonstrate an end-to-end NLP fine-tuning pipeline: data loading, tokenization, training with Hugging Face `Trainer`, evaluation (confusion matrix + per-class metrics), and deployment.

## Demo

Run `python app.py` after training to launch the interface locally, or deploy it to [Hugging Face Spaces](#deploying-to-hugging-face-spaces) for a public link.

## How it works

| | |
|---|---|
| **Model** | [`distilbert-base-uncased`](https://huggingface.co/distilbert-base-uncased) (66M params) |
| **Dataset** | [`dair-ai/emotion`](https://huggingface.co/datasets/dair-ai/emotion) — ~20K English Twitter messages, 6 emotion classes |
| **Task** | Single-label multi-class text classification |
| **Framework** | 🤗 Transformers `Trainer` API |
| **Demo** | Gradio `Interface` with live confidence scores per class |

## Quick start

```bash
git clone <your-repo-url>
cd <your-repo-name>
pip install -r requirements.txt

# Train (full run — a few minutes on a free Colab T4 GPU, longer on CPU)
python train.py

# Launch the demo using the model you just trained
python app.py
```

Open the local URL Gradio prints (usually `http://127.0.0.1:7860`).

### Quick smoke test (no waiting around)

Before committing to a full run, sanity-check the whole pipeline in under a minute:

```bash
python train.py --sample_size 500 --epochs 1
python app.py
```

### Useful flags

```bash
python train.py --epochs 5 --batch_size 32 --learning_rate 3e-5
```

Run `python train.py --help` for the full list.

## Deploying to Hugging Face Spaces

1. Push your trained model to the Hub so the Space doesn't need the (large) model folder committed to git:
   ```bash
   huggingface-cli login
   python train.py --push_to_hub --hub_model_id your-username/emotion-classifier
   ```
2. In `app.py`, set:
   ```python
   MODEL_SOURCE = "your-username/emotion-classifier"
   ```
3. Create a new Space on [huggingface.co/new-space](https://huggingface.co/new-space) (SDK: **Gradio**), and push `app.py` + `requirements.txt` to it.

## Possible extensions

- Swap in a different HF Hub dataset (e.g. `ag_news` for topic classification, or your own scraped/labeled data) — `train.py` only assumes `text`/`label` columns, so most changes are a one-line edit to `DATASET_NAME` and `LABEL_NAMES`.
- Try a larger backbone (`bert-base-uncased`, `roberta-base`) and compare accuracy vs. inference latency.
- Add per-example error analysis: pull the highest-confidence *wrong* predictions from the test set to see what the model consistently confuses (love vs. joy, surprise vs. fear are common ones).
- Wrap `predict()` in a small FastAPI service if you want a plain REST endpoint instead of (or alongside) the Gradio UI.
